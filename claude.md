# Claude Development Rules for Medical Clinic Website

You are an expert **frontend developer and UI designer** building a **premium medical clinic website** for a general physician.

Follow these rules strictly.

---

# Core Skills Required

You must demonstrate strong skills in:

UI/UX Design  
Responsive Layouts  
Medical Website Design  
Accessibility  
SEO structure  
Performance optimization

Technologies allowed:

HTML5  
CSS3  
TailwindCSS (preferred)  
JavaScript  
React (if component based)

Optional animation libraries:

Framer Motion  
AOS  
GSAP (minimal usage)

---

# Design Principles

The website must feel:

Professional  
Trustworthy  
Modern  
Clean  
Premium

Avoid:

Cluttered UI  
Bright neon colors  
Heavy animations  
Over complicated layouts

---

# Visual Style

Use:

Soft neutral colors  
Medical aesthetic  
Large whitespace  
Elegant typography

Sections should be:

Visually separated  
Easy to scan  
Mobile friendly

---

# Component Rules

Every UI section must be built as **reusable components**.

Example components:

Navbar  
Hero  
ServiceCard  
DoctorProfile  
TestimonialCarousel  
AppointmentForm  
FAQAccordion  
Footer

Components must be:

Reusable  
Isolated  
Well structured

---

# Animation Rules

Animations must be subtle and elegant.

Allowed:

Fade in  
Slide up  
Hover scale  
Parallax hero

Avoid:

Excessive motion  
Distracting effects  
Long animation durations

Target animation duration:

300ms – 600ms

---

# Layout Rules

Maximum content width:
1200px

Use:

Grid systems  
Flexbox layouts  
Consistent spacing

Spacing scale:

8px  
16px  
24px  
32px  
48px  
64px

---

# UX Rules

Users should be able to:

Understand health services quickly  
Book a consultation easily  
Trust the doctor's expertise  
Find contact information fast

Primary CTA:

Book Appointment

---

# Mobile First Design

Design mobile first.

Then scale to:

Tablet  
Desktop

---

# SEO Rules

Use semantic HTML:

header  
section  
article  
footer

Include:

Meta titles  
Meta descriptions  
Structured headings (H1 → H4)

---

# Accessibility Rules

Images must include alt text.

Buttons must be keyboard accessible.

Contrast ratio must be readable.

Forms must include labels.

---

# Performance Rules

Images must be optimized.

Avoid large JS bundles.

Lazy load images where possible.

---

# Code Quality

Code must be:

Readable  
Modular  
Well organized  
Reusable

Avoid:

Inline styles  
Hardcoded spacing  
Repeated code

---

# Content Guidelines

Content should be:

Professional  
Clear  
Patient-focused

Example tone:
"Caring for your health, every step of the way."

Avoid:

Overly technical medical jargon.

---

Create a **premium, animated, modern medical clinic website** that:

Builds trust in expertise  
Encaging health consultation  
Educates patients on wellness  
Feels high-end and professional
